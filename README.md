# Система за Управление на Турнири (Tournaments System)

## Описание
Това е индивидуален практически проект, разработен за курса **Spring Fundamentals** в СофтУни. Приложението представлява уеб платформа за управление на гейминг турнири и отбори. То позволява регистрация на потребители, създаване на отбори за различни игри, организиране на турнири и записване на мачове.

## Технологичен Стек (Tech Stack)
Проектът е изграден върху съвременна Java екосистема, спазвайки добрите практики за обектно-ориентиран дизайн и архитектура:
* **Език:** Java 17
* **Framework:** Spring Boot 3.4.0
* **База данни:** MySQL (Релационна база данни)
* **Достъп до данни:** Spring Data JPA / Hibernate
* **Сигурност:** Session-based authentication & Spring Security Crypto (Bcrypt хеширане на пароли)
* **Frontend:** Spring MVC + Thymeleaf (сървърно рендиране на HTML с Bootstrap 5)
* **Build Tool:** Maven

## Ключови Функционалности (Features)
Приложението покрива следните бизнес изисквания и CRUD операции:

1. **Автентикация и Оторизация (Access Control):**
    * Регистрация и Вход на потребители със сесийно управление.
    * Ролеви модел (Role-based access): първият регистриран потребител автоматично получава роля `ADMIN`, а останалите `USER`.
    * Защитени маршрути: само логнати потребители имат достъп до вътрешните страници.

2. **Управление на Отбори (Teams):**
    * Създаване на нов отбор с валидация на данните (име и игра).
    * Визуализация на всички активни отбори.
    * Изтриване на отбор (достъпно само за потребители с роля `ADMIN`).

3. **Управление на Турнири (Tournaments):**
    * Създаване на нов турнир.
    * Свързване на отбори към турнири чрез релация Many-To-Many в базата данни.
    * Изтриване на турнири.

4. **Глобална Обработка на Грешки (Error Handling):**
    * Валидация на потребителския вход при всяка форма (Server-side validation) с визуализация на грешките в реално време.
    * Прихващане на бизнес изключения (напр. дублирано име на отбор) чрез `@ControllerAdvice` и пренасочване към специализирана страница за грешки.
